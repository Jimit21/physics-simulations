# Project Rules

1. No Emojis: Under no circumstances should emojis be used in source code, documentation, markdown files, social media posts, or any other project-related content.
2. Professional but Persuasive Tone: Do not use enthusiastic, flowery, or standard "AI-generated" phrasing.
3. Showmanship and Storytelling: Marketing, posts, and external communications must have aggressive showmanship and compelling storytelling, akin to a Saul Goodman advertisement. It must not be boring. Hook the reader, present a stark problem, and offer an undeniable solution.
4. Obfuscation: All core simulation logic (JavaScript) must remain obfuscated to prevent unauthorized copying.
5. Privacy: Do not mention specific school names or current employers in any public-facing code or documentation.
